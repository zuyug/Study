모바일 기준 핵심 정리

GitHub 파일 화면이 아니라 Pages 사이트로 봐야 합니다.

GitHub repo는 노트의 원본 저장소입니다. 모바일에서 읽기 좋은 화면은 GitHub Pages로 배포된 사이트입니다. 앞으로 공부할 때는 Pages 주소를 북마크하거나 홈 화면에 추가해서 보는 것이 맞습니다.

정답 주소: https://zuyug.github.io/Study/

원본 저장소: https://github.com/zuyug/Study

1. 화면을 구분해야 합니다

읽을 때 쓰는 화면

GitHub Pages입니다. 글자 크기, 목차, 검색, 모바일 레이아웃이 읽기용으로 맞춰져 있습니다.

Study Pages 열기

관리할 때 쓰는 화면

GitHub repo 파일 화면입니다. 소스 파일을 저장하고 확인하는 곳이라 모바일 독서용으로는 불편합니다.

Study repo 열기
모바일에서 GitHub 앱이나 github.com 파일 화면으로 들어가면 가독성이 떨어지는 것이 정상입니다. 그 화면은 책장이 아니라 창고에 가깝습니다. 실제로 읽을 때는 Pages 주소를 쓰면 됩니다.

2. 모바일에서 보는 순서

  1. 휴대폰 브라우저에서 Pages 주소를 엽니다.
    주소창에 https://zuyug.github.io/Study/를 입력하거나 이 보고서의 링크를 누릅니다.
  2. 왼쪽 위 메뉴 버튼을 누릅니다.
    모바일에서는 왼쪽 사이드바가 접혀 있습니다. 상단의 메뉴 버튼을 누르면 경제, 코딩, AI, 생활, 복습 카테고리가 나옵니다.
  3. 검색 아이콘을 눌러 단어로 찾습니다.
    예를 들어 PER, 재무제표, ROE, Python 같은 단어로 검색합니다.
  4. 읽고 싶은 노트를 엽니다.
    현재 예시는 경제 > PER/PBR/ROE입니다.
  5. 복습이 필요하면 복습 메뉴를 봅니다.
    Anki용 CSV 파일은 복습 > Anki 카드에서 확인합니다.

3. 북마크와 홈 화면 추가

iPhone Safari

  1. Study Pages를 엽니다.
  2. 하단 공유 버튼을 누릅니다.
  3. 홈 화면에 추가를 선택합니다.
  4. 이름을 Study로 저장합니다.

Android Chrome

  1. Study Pages를 엽니다.
  2. 오른쪽 위 점 세 개 메뉴를 누릅니다.
  3. 홈 화면에 추가를 선택합니다.
  4. 이름을 Study로 저장합니다.

이렇게 해두면 GitHub repo를 찾아 들어갈 필요 없이 앱처럼 바로 열 수 있습니다.

4. 어떤 주소를 써야 하나?

목적 사용할 주소 설명
공부 노트 읽기 Pages 메인 모바일 가독성이 가장 좋습니다. 앞으로 기본으로 이 주소를 쓰면 됩니다.
PER/PBR/ROE 바로 보기 PER/PBR/ROE 노트 현재 첫 번째 경제 노트입니다.
원본 파일 확인 GitHub repo Codex/Claude가 수정하고 저장하는 원본입니다. 읽기용으로는 권장하지 않습니다.
복습 카드 확인 Anki 카드 페이지 노트에서 뽑은 질문/답변 복습 카드 목록입니다.

5. 앞으로 노트추가를 하면 어떻게 되나?

앞으로 사용자가 질문한 뒤 노트추가라고 말하면 아래 흐름으로 처리하는 것이 목표입니다.

  1. 방금 답변 내용을 초보자용 설명으로 다시 정리합니다.
  2. 경제, 코딩, AI, 생활 중 하나의 카테고리를 고릅니다.
  3. 태그를 붙입니다. 예: 주식, 재무제표, Python, 프롬프트
  4. Markdown 원본 파일을 docs/ 아래에 추가합니다.
  5. Anki 복습 카드도 같이 추가합니다.
  6. MkDocs 빌드를 검증합니다.
  7. GitHub에 푸시하면 Pages 사이트가 자동 배포됩니다.

6. GitHub 파일 화면을 써야 하는 경우

GitHub repo 화면은 아예 쓸 필요가 없는 것은 아닙니다. 다만 용도가 다릅니다.

  • 원본 Markdown 파일이 실제로 올라갔는지 확인할 때
  • Codex/Claude가 어떤 파일을 수정했는지 확인할 때
  • 커밋 기록이나 Actions 배포 상태를 볼 때
  • 직접 파일을 수정해야 할 때
정리된 글을 읽는 목적이라면 GitHub 파일 화면으로 들어가지 말고 Pages 주소로 들어가야 합니다.
